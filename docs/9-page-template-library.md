# Page Template Library

## Purpose

This document defines standard page templates for future Velu website development.

Templates help preserve page purpose, messaging hierarchy, conversion paths, SEO structure, and consistency across public website pages.

## Scope

This library applies to future homepage updates, service pages, resource pages, lead magnet landing pages, and future blog content.

It does not approve new pages by itself. New pages must still align with approved website strategy, content standards, SEO strategy, schema standards, and roadmap status.

## Homepage Template

Required sections:

- Hero
- Outcome Framework
- Pain Points
- Service Introduction
- Trust Building
- Lead Magnet
- Reviews
- CTA

The homepage template should establish Velu as a nonprofit accounting and advisory firm, introduce the outcome framework, connect services to outcomes, and direct qualified visitors toward the primary get-started conversion path.

Reference:

- [Homepage Framework](5-homepage-framework.md)

## Service Page Template

Required sections:

- Hero
- Outcome Connection
- Problem
- How Velu Helps
- Service Pillars
- Outcomes
- Reviews
- CTA

Service pages should lead with nonprofit leadership outcomes before service deliverables. They should connect Outcome -> Capability -> Service -> Deliverable and should follow the service-page schema and review standards when schema is implemented.

Service page hero sections should include the Velu Financial Growth Framework hero visual when applicable. The highlighted framework pillar should align with the service page's primary outcome. Additional service-specific images should support the body sections, not replace the framework hero unless an exception is approved.

Reference:

- [Service Page Framework](4-service-page-framework.md)

## Resource Page Template

Required sections:

- Topic Introduction
- Problem
- Educational Content
- Action Steps
- Related Services
- CTA

Resource pages should answer practical nonprofit financial management questions and support approved topic clusters. They should connect educational content to relevant service pages without forcing a sales message where education is the primary purpose.

## Lead Magnet Landing Page Template

Required sections:

- Lead Magnet Introduction
- Benefits
- Download Form
- Related Services
- CTA

Lead magnet landing pages should clearly explain the resource, who it helps, and what practical value it provides. They should support authority building and may function as secondary conversion paths.

## Future Blog Template

Required sections:

- Topic Introduction
- Educational Content
- Related Resources
- Related Services
- CTA

Future blog content should support approved topic clusters and link to relevant services or resources where useful. Blog templates should not be used to create generic accounting articles that weaken nonprofit positioning.

## Future AI Agent Instructions

- Use templates unless an approved exception exists.
- Preserve page purpose before modifying structure.
- Do not remove CTA paths without approval.
- Do not create new page templates unless a recurring page type requires one.
- Keep templates aligned with the website messaging framework, SEO framework, and schema standards.
- Distinguish template requirements from optional page enhancements.
