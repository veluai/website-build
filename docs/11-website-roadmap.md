# Website Roadmap

## Purpose

This document defines the strategic roadmap for future Velu website development.

The roadmap tracks website initiatives by status so future work can distinguish completed standards, active work, planned initiatives, and proposed ideas.

## Scope

This roadmap applies to public website strategy, framework documentation, page development, schema work, SEO expansion, content expansion, image inventory work, and future website operating standards.

It does not approve internal systems, CRM workflows, client portals, marketing campaigns, or non-website operations.

## Roadmap Philosophy

The roadmap tracks approved website initiatives and separates them by status:

- Complete
- Active
- Planned
- Proposed

Status definitions:

- Complete: Work has been created or implemented and is part of the current Website Build operating framework or known website state.
- Active: Work is currently underway or identified as an immediate active initiative.
- Planned: Work has strategic value and may proceed when prioritized and approved for execution.
- Proposed: Work is a possible future opportunity and should not be treated as approved work.

## Complete

Completed items:

- Homepage Framework
- Service Page Framework
- Schema Standards
- Image SEO Standards
- SEO Framework
- Messaging Framework
- Current homepage schema
- Outsourced Accounting page schema
- Client Advisory Services page schema

Completed framework documents should remain source documents for future website work unless updated through the revision history process.

## Active

Active initiatives:

- Grant Advisory & Compliance page development
- Website framework expansion

Active initiatives may still require copy, design, schema, image, validation, and approval work before being considered complete.

## Planned

Planned initiatives:

- Additional service pages
- Resource pages
- Lead magnet expansion
- Topic cluster expansion
- Image inventory population
- Page template refinement

Planned initiatives should be developed in alignment with existing framework documents and should not be published without appropriate review.

## Proposed

Proposed initiatives:

- Blog architecture
- Resource library
- Additional educational assets
- Future SEO initiatives

Proposed items are not approved work. They require strategic review before moving into Planned status.

## Roadmap Governance

Only approved initiatives move from Proposed to Planned.

Tyler Wilcox is final approver.

Roadmap updates should be made when major website initiatives are completed, newly approved, paused, replaced, or archived. Material decisions should also be reflected in the relevant framework or standards document when applicable.

## Future AI Agent Instructions

- Do not treat Proposed items as approved work.
- Distinguish roadmap status clearly.
- Update roadmap when major website initiatives are completed.
- Do not move items between statuses without approval or clear source support.
- Keep roadmap language strategic and durable rather than temporary project notes.
- Cross-reference standards when roadmap items affect schema, SEO, content, images, or page templates.
