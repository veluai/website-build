# Image Library Inventory

## Purpose

This document establishes governance and inventory standards for Velu website imagery.

This file is not intended to become a complete image database today. It defines the structure future maintainers should use when tracking approved website images, page usage, image purpose, SEO metadata, and URL verification.

## Scope

This inventory standard applies to public website imagery, including homepage images, service page images, framework graphics, service pillar images, collaboration images, lead magnet images, logo files, review or testimonial images, and future blog or resource images.

It does not govern source design files, internal creative drafts, or non-website image assets unless those assets are approved for public website use.

## Image Inventory Philosophy

The image inventory exists to:

- Track approved website images
- Track image purpose
- Track page usage
- Track SEO metadata
- Reduce duplication

Images should support page messaging, accessibility, trust, and nonprofit-specific positioning. They should not be added only for decoration when they create maintenance burden or weaken page clarity.

## Inventory Fields

Future image inventory entries should use this table structure:

| Image Name | Category | Page | Purpose | Alt Text Status | URL Verified | Owner |
| --- | --- | --- | --- | --- | --- | --- |

Field guidance:

- Image Name: Clear internal name for the image.
- Category: Image category such as hero, framework, service pillar, lead magnet, collaboration, logo, or testimonial.
- Page: Page where the image appears.
- Purpose: Why the image exists on the page.
- Alt Text Status: Draft, approved, implemented, or not applicable.
- URL Verified: Yes, no, pending, or not applicable.
- Owner: Person responsible for approval or maintenance.

## Current Known Image Categories

Current known image categories include:

- Homepage Hero
- Homepage Collaboration
- Homepage Lead Magnet
- Outsourced Accounting Pillars
- Client Advisory Pillars
- Logo

These categories should be populated with specific image records when image URLs, alt text, and page usage are ready to be inventoried.

## Image Governance Rules

- Images should have a defined purpose.
- Images should support page messaging.
- Alt text should follow Image SEO Standards.
- URLs should be verified before publication.
- Images used in schema should reference actual image URLs.
- Image choices should reinforce nonprofit accounting and advisory positioning.
- Duplicate or near-duplicate images should be avoided unless they serve a clear page purpose.

## Future AI Agent Instructions

- Do not invent image contents, filenames, or URLs.
- Do not mark URL verification complete unless the URL has been checked.
- Do not mark alt text implemented unless it has been placed in the website image settings or HTML.
- Use this document to structure inventory records, not to create unsupported image records.
- Keep image inventory aligned with page purpose, SEO standards, and schema standards.
