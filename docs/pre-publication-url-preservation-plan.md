# Pre-Publication URL Preservation Plan

## Purpose

This document defines the focused pre-publication process for deciding what to preserve from the current WordPress site before launching the simplified GoHighLevel replacement site.

This is not a full migration plan. It is not a full content repurposing plan. It is a focused decision framework for classifying current WordPress URLs, resources, posts, and lead magnets before publication.

## Site Transition Context

- Current platform: WordPress
- New platform: GoHighLevel
- Transition type: rebuild, not migration
- New site goal: simplified, conversion-focused, nonprofit-leader-focused website
- Main conversion goal: Book Your Discovery Call
- New site publication status: pending
- Live validation status: pending publication

Velu is replacing its current WordPress website with a simplified GoHighLevel website. The new site should prioritize nonprofit CEOs, executive directors, COOs, board members, and other decision-makers.

## Audience Priority

Velu's primary launch-site audience is:

- nonprofit CEOs
- executive directors
- COOs
- board members
- board treasurers or finance committee leaders
- decision-makers responsible for financial clarity, accountability, funding, and sustainable growth

The old site may have attracted lower-fit audiences, including:

- nonprofit accountants
- bookkeepers
- DIY treasurers
- template seekers
- students
- technical accounting researchers

Audience fit has priority over raw traffic preservation.

## Core Preservation Rules

`Preserve old URLs only when they have SEO value and support Velu’s current buyer-focused positioning. Do not preserve traffic for its own sake if the likely visitor is a practitioner, DIY template seeker, student, or wrong-fit lead.`

`If the page does not support a nonprofit leader’s decision to trust Velu, book a call, or understand a core service, it does not belong on the main site at publication.`

- Preserve only what supports trust, discovery-call intent, core service understanding, or buyer confidence.
- Do not recreate the old site’s complexity.
- Do not move technical resource content into the main launch site by default.
- Do not let old SEO traffic override the new positioning strategy.

## URL Classification Categories

| Category | Use When | Launch Action |
| --- | --- | --- |
| Preserve / Redirect | URL has SEO value and supports nonprofit leader or buyer intent | 301 redirect to the closest relevant new page |
| Archive for Funnel | URL has SEO or content value and is a realistic future funnel, lead magnet, nurture, or repurposing candidate | Archive for future funnel, lead magnet, nurture, or repurposing project |
| Archive | URL should be retained for internal reference only, but is not part of the launch site and is not currently planned for future funnel use | Archive for retention only |
| Discard | URL has low SEO value, low strategic fit, and no meaningful retention need | No special preservation unless technically required |
| Review | Value, audience fit, retention need, or future use is unclear | Manual decision before publication |

## Evaluation Criteria

### Test 1: SEO Value

A URL has possible SEO value when it has one or more of:

- meaningful organic traffic
- backlinks
- brand relevance
- current rankings
- clear search demand
- a logical new destination

### Test 2: Audience and Strategy Fit

A URL supports the launch site when it helps a nonprofit leader:

- trust Velu
- book a discovery call
- understand a core service
- understand financial clarity, accountability, funding confidence, or sustainable growth
- see Velu as a strategic financial partner

`A URL should generally be preserved only when it passes both tests.`

## Content Likely To Preserve

Likely preservation candidates include:

- old homepage
- old About page
- old contact or schedule pages
- old bookkeeping/accounting service pages, if mapped to Outsourced Accounting
- old CFO/advisory/consulting pages, if mapped to Client Advisory
- old grant-related pages, if mapped to Grant Advisory & Compliance
- old tax or Form 990 service pages, if mapped to Tax Services
- leadership-oriented articles with clear buyer fit
- pages with strong brand relevance

## Content Likely To Archive For Future Funnels

Likely archive-for-funnel candidates include:

- templates
- checklists
- how-to accounting resources
- QuickBooks setup content
- chart of accounts resources
- functional expense resources
- technical Form 990 explainers
- treasurer or bookkeeper tools
- practitioner-focused guides

Use `Archive for Funnel` only when the content has realistic future marketing value. These assets may be valuable later as:

- funnels
- gated lead magnets
- nurture sequences
- segmented campaigns
- executive-facing repurposed content

They are not part of the launch-site rebuild. Pre-publication repurposing is intentionally out of scope.

Use `Archive` when content is being retained only for reference and is unlikely to be reused. Technical resources may fall into either category depending on owner decision.

## Content Likely To Archive For Retention

Some URLs may be worth retaining internally but should not be treated as future marketing assets.

Likely archive-for-retention candidates include:

- old process pages no longer aligned with the simplified site
- old service/resource pages that may be useful for reference but are not planned for launch or funnels
- app/subdomain URLs not part of the public launch-site rebuild
- content with limited reuse potential but possible historical value
- content the owner wants retained but does not want prioritized for future funnel development

`Archive means retain for reference only. It does not mean preserve for launch, redirect for SEO, or prioritize for future funnel development.`

## Content Likely To Discard

Likely discard candidates include:

- outdated posts
- low-value announcements
- thin content
- duplicate content
- old pages with no clear new destination
- pages that attract wrong-fit traffic and have little SEO value
- content that conflicts with the simplified positioning

## Focused Audit Outputs

The focused audit should produce:

1. URL inventory
2. Preserve/redirect list
3. Archive-for-funnel list
4. Archive-for-retention list
5. Discard list
6. Review list
7. Launch redirect map
8. Post-publication redirect and 404 monitoring notes

## Out of Scope Before Publication

These items are intentionally out of scope before publication:

- full blog migration
- full resource migration
- rebuilding old lead magnets into the new site
- repurposing technical content into executive-facing funnels
- redesigning archived resources
- creating new funnel assets
- broad keyword expansion
- preserving traffic from wrong-fit audiences for its own sake
- repurposing archive-for-retention items before publication
- turning archive-for-retention items into launch-site pages unless separately approved
- treating archive-for-retention items as future funnel priorities unless later reclassified

## Post-Publication Monitoring

After publication, Velu should monitor:

- 404 errors
- redirect behavior
- sitemap submission
- indexing status
- top old URLs
- organic traffic changes
- discovery-call conversions
- form submissions
- wrong-fit vs right-fit lead quality
- Search Console coverage issues
